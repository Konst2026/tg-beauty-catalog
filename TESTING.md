## 1. Что тестируем

**BeautyBook** — Telegram Mini App (TMA): маркетплейс бьюти-мастеров.

**Две роли в одном приложении:**
- **Клиент** — смотрит каталог мастеров, записывается на услугу
- **Мастер** — ведёт профиль: фото, услуги, расписание

**Где живёт:**
| Среда | URL |
|---|---|
| Production (Vercel) | `https://tg-app-mu-two.vercel.app` |
| Telegram-бот | `@BeautyAppBook_bot` |
| GitHub | `https://github.com/Konst2026/tg-beauty-catalog` |

**API-эндпоинты (Vercel Serverless Functions):**
| Метод | URL | Назначение |
|---|---|---|
| `POST` | `/api/bot` | Webhook Telegram-бота |
| `GET` | `/api/masters` | Список всех мастеров с сервера |
| `POST` | `/api/masters` | Сохранить/обновить профиль мастера |
| `DELETE` | `/api/masters` | Удалить профиль мастера |

## 2. Архитектура за 5 минут

Vanilla JS SPA — без фреймворков, без сборки. Порядок загрузки скриптов в `index.html` критичен:

```
src/api/data.js        → данные, MASTERS[], геттеры, localStorage, fetch к API
src/screens/client/    → renderCatalog(), renderMasterProfile(), ...
src/screens/master/    → renderMasterDashboard(), renderMasterProfileEdit(), ...
src/utils/events.js    → bindScreenEvents() — DOM-события после каждого рендера
src/app.js             → state{}, navigate(), init()
```

**Хранилище данных мастера:**
- `localStorage` ключ: `bb_master_m_{telegram_user_id}` — хранит весь профиль включая base64-фото
- GitHub Gist `cf2b26e11667c82f27d37636a8ee8dff` (через `/api/masters`) — хранит текстовые данные БЕЗ фото
- При загрузке: сначала `loadMasterFromStorage()` (фото), потом `loadMastersFromServer()` (данные с сервера)

**TG SDK мок (при открытии в браузере):**
```javascript
// src/app.js:27
tg.initDataUnsafe.user = { first_name: 'Алина', last_name: 'Петрова', username: 'alina_p', id: 12345 }
```
В браузере `state.myMasterId = 'm_12345'`. В реальном Telegram — `'m_' + tg.initDataUnsafe.user.id`.

## 3. Запуск локально (без Telegram)

### Шаг 1. Клонировать репозиторий

```bash
git clone https://github.com/Konst2026/tg-beauty-catalog.git
cd tg-beauty-catalog/tg-app
```

### Шаг 2. Запустить любой HTTP-сервер

Приложение — статика. Открыть `index.html` через `file://` НЕ работает из-за политики CORS.

**Вариант A — Python (если установлен):**
```bash
python -m http.server 8080
# Открыть: http://localhost:8080
```

**Вариант B — Node.js:**
```bash
npx serve . -p 8080
# Открыть: http://localhost:8080
```

**Вариант C — VS Code Live Server:**
Установить расширение Live Server → правая кнопка на `index.html` → «Open with Live Server».

### Шаг 3. Проверить загрузку

В браузере открыть `http://localhost:8080`. Ожидаемое поведение:
1. 1,5 сек — экран splash («BeautyBook» + progress bar)
2. Если `localStorage.bb_onboarding_done` не установлен → экран онбординга
3. Иначе → каталог мастеров

> **Важно:** API-вызовы (`/api/masters`) будут возвращать ошибку 404 при локальном запуске — это нормально. Данные из MASTERS[] в коде загрузятся всегда. Тестирование API производится только на деплое.

## 7. Тест-план: API (через curl / Postman)

**Base URL:** `https://tg-app-mu-two.vercel.app`

### TC-18 GET /api/masters

```bash
curl https://tg-app-mu-two.vercel.app/api/masters
```

**Ожидаемый результат:** HTTP 200, массив JSON с профилями мастеров (может быть пустым `[]`).  
**Формат каждого мастера:**
```json
{
  "id": "m_12345",
  "name": "Имя Фамилия",
  "specialty": "...",
  "categoryId": "nails|lashes|brows|hair",
  "city": "...",
  "priceFrom": 1200,
  "availableToday": false,
  "services": [],
  "gallery": [],
  "promo": ""
}
```

### TC-19 POST /api/masters (создание/обновление)

```bash
curl -X POST https://tg-app-mu-two.vercel.app/api/masters \
  -H "Content-Type: application/json" \
  -d '{"id":"test_master","name":"Тест Мастер","categoryId":"nails","services":[],"gallery":[]}'
```

**Ожидаемый результат:** HTTP 200, `{ "ok": true }`.

### TC-20 POST /api/masters — ошибка без id

```bash
curl -X POST https://tg-app-mu-two.vercel.app/api/masters \
  -H "Content-Type: application/json" \
  -d '{"name":"Без айди"}'
```

**Ожидаемый результат:** HTTP 400, `{ "error": "Missing id" }`.

### TC-21 DELETE /api/masters

```bash
curl -X DELETE https://tg-app-mu-two.vercel.app/api/masters \
  -H "Content-Type: application/json" \
  -d '{"id":"test_master"}'
```

**Ожидаемый результат:** HTTP 200, `{ "ok": true }`. Мастер `test_master` пропал из GET /api/masters.

### TC-22 CORS

```bash
curl -X OPTIONS https://tg-app-mu-two.vercel.app/api/masters \
  -H "Origin: https://example.com" \
  -H "Access-Control-Request-Method: POST" -v
```

**Ожидаемый результат:** HTTP 200, заголовки:
- `Access-Control-Allow-Origin: *`

## 9. Известные ограничения (не баги)

| Ограничение | Причина | Поведение |
|---|---|---|
| Фото не синхронизируются между устройствами | base64-фото слишком большие для GitHub Gist | На чужом устройстве вместо фото — цветной градиент |
| Записи хранятся только в памяти сессии | Нет backend БД | После перезагрузки страницы `MY_BOOKINGS` сбрасывается к демо-данным |
| Слоты генерируются псевдослучайно | Нет реального расписания в БД | `generateSlots()` использует алгоритм `(idx + seed) % 3 !== 0`, убирает ~30% слотов |
| Расписание мастера не сохраняется на сервер | Нет endpoint для расписания | Изменения в расписании хранятся только в `MASTER_DASHBOARD` в памяти |
| При отсутствии `GITHUB_TOKEN` на сервере | env var не задана | `/api/masters` возвращает `[]` без ошибки (silent fallback) |

## 11. Как воспроизвести конкретный экран напрямую

Все экраны доступны через навигацию. Для быстрого доступа через DevTools Console:

```javascript
// Онбординг
localStorage.removeItem('bb_onboarding_done'); location.reload();

// Сразу каталог
localStorage.setItem('bb_onboarding_done','1'); navigate('catalog',{},'none');

// Профиль мастера (режим клиента — посмотреть мастера m2)
navigate('master-profile', { selectedMasterId: 'm2' });

// Кабинет мастера
state.isMasterMode = true; navigate('master-profile-edit',{},'none');

// Снова режим клиента
state.isMasterMode = false; navigate('catalog',{},'none'); updateTabBar('catalog');

// Оффер-модал
localStorage.removeItem('bb_offer_shown'); navigate('catalog',{},'none');
```
